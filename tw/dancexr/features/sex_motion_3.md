# ./motion/proc/sm3

適用於男性和女性角色成對的程序化成對性動作。女性側負責生成搖擺、抽插節奏、接觸框架和興奮狀態；男性側則綁定到此接觸框架上，確保雙方保持鎖定，而不是像兩個獨立動畫一樣漂移。


## 上半身擺動與抽插

**上半身擺動動畫（Sway Motion）** 負責在整個循環上層疊的軀幹擺動風格，而 **成對性動作（Sex Motion）** 則控制著深入的節奏、移動距離和速度反應。請將前一個動作視為視覺風格，而將後一個動作視為潛在的機械驅動層。擺動過大可以增強重量感，但如果它相對於抽插循環過大，則會讓成對動作看起來鬆散，而非物理上連接。


## 接觸與反應

**接觸平滑（Contact Smoothing）** 主要適用於男性角色：它過濾了由女性驅動的接觸框架，防止骨盆的細微抖動傳染給夥伴。當綁定感覺雜訊過多時，請提高此值；當男性開始落後於動作時，請降低此值。

**反應（Reaction）** 會根據抽插伸展來彎曲脊椎。**速度（Speed）** 和 **阻尼（Damping）** 決定了彈簧追隨的速度；**彎曲（Bend）**、**髖部/軀幹比率（Hip/Torso Ratio）** 和 **混合（Blend）** 則決定了動作在身體哪個區域呈現。將彎曲推得太遠，會讓循環看起來像舞台劇，而不是具反應性。


## 興奮與高潮

**高潮（Orgasm）** 模組增加了一層額外的效果，可以加速動作、融合姿勢，並驅動顫抖和臉部強度。在 *物理（Physical）* 模式下，峰值是由抽插刺激觸發的，因此 **高潮敏感度（Orgasm Sensitivity）**、**音樂補償（Music Compensation）** 和 **興奮曲線（Arousal Curve）** 控制了積累的容易程度。在 *確定（Determined）* 模式下，則改為按節拍計數，當你希望有可預測的音樂節奏時，效果更佳。

一旦觸發，**顫抖強度（Shaking Intensity）**、**顫抖速度（Shaking Speed）**、**顫抖阻尼（Shaking Damping）** 和 **顫抖頻率（Shaking Frequency）** 就定義了高潮的感覺。**變化度（Variety）** 增加了每個高潮之間的漂移，使顫抖不那麼重複，而 **射精機會（Ejaculation Chance）** 改變了姿勢是內收塌陷還是會發展到射精分支。
使用 **測試（Test）** 時進行調校，這樣你就可以在不需要即時刺激的情況下看到完整的循環。


## 角色姿態對齊

**女性姿勢（Female Pose）** 和 **男性姿勢（Male Pose）** 在應用程序化層之前，用於為每個角色設定基礎身體布局。在女性側，**陰部向上/向下（Pussy Up / Down）**、**陰部前方/後方（Pussy Front / Back）** 和 **陰部角度（Pussy Angle）** 用於移動用來發佈綁定框架的接觸錨點。在這裡進行的小調整，通常比強行施加大的姿勢偏移更好，因為錨點會影響對齊和穿透方向兩方面。

在男性側，**彎曲陰莖（Bend Penis）** 會將骨骼鏈條朝向解決目標彎曲。除非模型需要幫助到達接觸點，否則保持此值較低，否則修正看起來會比動作本身更明顯。


## 表情

**臉部（Facial）** 將程序化強度映射到眉毛、眼瞼和嘴部的形變上。如果模型本身已經有完整的臉部動畫，請讓此模組保持低調；如果希望即使在遠距離也能清楚看出情緒變化，則可以適度加強。

# 組件細項

## 上半身擺動動畫

這是一個可重複使用的動作模式生成器，用於生成循環的身體擺動和位置漂移。它可以隨機使用內建模式、隨機輪換已儲存的預設，或直接公開底層曲線供手動塑形。


### 模式來源


**模式（Mode）** 決定曲線的來源。*隨機（Random）* 從內建模式庫中抓取，*隨機預設（Random Preset）* 輪換你儲存的預設，*手動（Manual）* 則直接公開底層動作曲線。**種子（Seed）** 固定了隨機序列，使相同的模式順序重複；當你想要新的變化而無需重新設計曲線時，請更改此值。


### 時間和強度


**每個群組動作數（Moves Per Group）** 控制生成器切換到下一個動作短句的頻率。**速度（Speed）** 縮放播放速率，而 **使用音訊（Use Audio）** 則讓動作隨著音樂的節奏而非固定強度呼吸。**延伸範圍（Extent）** 是可自動更新的，因此當你想讓另一個系統隨時間推動動作變得更大或更小時，它是最佳的控制參數。


### 過渡與阻尼


**過渡（Transition）** 柔化了短句之間的交接；低值會使動作突然跳到下一個概念，高值使其更流暢，但也可能模糊了單個模式的特色。**阻尼（Damping）** 控制了驅動骨架在方向、水平擺動和垂直擺動上追趕的速度，這通常是使編舞精確度與漂浮感之間的區別所在。

### 動作 X


用於控制程序化運動的曲線函數。

### 動作 Z


用於控制程序化運動的曲線函數。


## 成對性動作

這是一個可重複使用的彈簧驅動抽插控制器。一個塑形後的驅動曲線推動一個質量點，第二個質量點跟隨其後，兩者之間的間隙就成為成對動作系統使用的受控位移。這使得循環感覺更具彈性，而不是像原始的正弦波。


### 節奏和移動


**延伸範圍（Extent）** 設定最大的移動距離。**自動強度（Auto Intensity）** 可以根據目前的音樂音量縮放這些移動，而 **自動 BPM（Auto BPM）** 和 **速度（Speed）** 控制了循環的運行速度。
當你需要一致的節奏時，請使用手動速度；當動作需要與原聲帶一起呼吸時，請啟用音訊驅動的控制。


### 驅動形狀


**頂部持續時間（Top Duration）**、**底部持續時間（Bottom Duration）** 和 **斜率平衡（Slope Balance）** 用於塑形理想的循環，然後彈簧才會對其產生反應。較長的頂部創造了維持的延伸，較長的底部創造了更明顯的重置，而斜率平衡則在推進和回程的動作間重新分配了時間。在這裡，你定義了動作是感覺有力、平穩還是挑逗。


### 彈簧反應


**碰撞距離（Collision Distance）** 設定了兩個彈簧質量之間的靜止分離距離。**彈簧 A（Spring A）**、**阻尼 A（Damping A）**、**彈簧 B（Spring B）**、**阻尼 B（Damping B）** 和 **靜態彈簧（Rest Spring）** 決定了每個質量點跟隨驅動的緊密程度，以及結果中殘留的超調或柔和度。較硬的數值感覺更機械化；較軟的數值感覺更重，但在循環快速時可能會變得鬆散。


### 可視化


**可視化曲線（Visualize Curve）** 在場景中繪製了目標和彈簧反應，這樣你就可以在沒有單憑身體動作的情況下調整其形狀。這是一個輔助設置工具，而不是你在正常使用時會保留的功能。


## 表情

將單一動作強度值映射到臉部形變，使表情可以在抽插循環、興奮累積或高潮峰值中逐步開啟。這是一個輕量級驅動器：它不會自己產生表情，而是重新映射現有的形變及其最小/最大範圍。


### 形變選擇


**眉毛形變（Eyebrow Morph）**、**眼瞼形變（Eyelid Morph）** 和 **嘴部形變（Mouth Morph）** 用於選擇接收驅動值的表情通道。選擇從中性到具表現力的形變，否則即使範圍很大，動作也會顯得虛弱。


### 輸出範圍


每個範圍都設定了驅動值從 0 到 1 移動時寫入的最小值和最大值。較窄的範圍使臉部看起來微妙，並疊加於其他動畫上；較寬的範圍使動作顯得更為明顯，但對於具有強烈形變的模型的動畫可能會出現截斷或衝突。


## 男性姿勢

一個可重複使用的角色姿勢塊，用於在應用動作層之前佈置角色。它結合了身體對齊、手部姿勢和腿部姿勢控制，讓程序化系統可以從一個穩定的已創作姿勢開始，而不是與模型預設的靜止姿勢抗衡。


### 身體設置


當主機功能啟用身體控制時，**方向（Orientation）**、**彎曲 X（Bend X）**、**彎曲 Y（Bend Y）**、**扭曲（Twist）**、**頭部旋轉（Head Rotation）**、**身體位置（Body Position）** 和 **身體旋轉（Body Rotation）** 用於建立基礎軀幹布局。請先用這些進行粗略的對齊。在不良的基礎姿勢上堆疊的大的程序化偏移，通常比使用更小的動畫修正進行乾淨佈置的姿勢效果更差。


### 手部

**手部（Hands）** 塊決定了手部姿勢是否啟用，以及兩側是否保持對稱。這在需要快速、鏡像化的姿勢進行大範圍佈景，或需要與夥伴或道具進行更自然的互動的非對稱姿勢時非常有用。


### 腿部


**腿部（Legs）** 塊對下半身姿勢做同樣的操作。當此功能應感覺居中且易於重定向時，請保持腿部對稱；當你需要體重轉移或更具角色特色的站姿時，請打破對稱性。由於許多動作系統都建立在這種姿勢之上，小腿部編輯可能會對平衡和接觸產生巨大的下游影響。
