# Release 2026.5

## 推出 DanceXR Operator
DanceXR Operator 是 DanceXR 專用的本地 AI 後端。它會以本地伺服器的形式和遊戲一起運作，把語音合成與大型語言模型聊天整合在統一 API 背後，讓 DanceXR 能在不依賴額外雲端服務的情況下提供 AI 對話、角色扮演與角色語音功能。

在實際使用上，Operator 正是這次發布中新版 AI 聊天架構的基礎。它被設計為直接在你自己的遊戲 PC 上運作，在本地管理模型與語音流程，為更長時間、且更貼近場景的角色互動提供更可靠的底層支撐。

### AI 聊天重構
AI 聊天現在已圍繞角色行為模擬的 agentic 工作流程重新打造。使用 Operator 作為專用本地後端後，它可以在標準遊戲設備上運行，而不需要額外的伺服器機器或雲端連線。

- **環境感知**：新的聊天系統會更早讀取場景資訊，也會更穩定地使用這些資訊，因此角色能根據目前位置、時間、燈光、舞台配置等內容做出回應，而不是脫離場景地對話。
- **記憶與持續性**：角色現在能更穩定地記住最近互動、關係狀態與關鍵事件。多輪歷史、意圖追蹤、會話持久化與記憶壓縮共同作用，讓對話更容易延續，而不是不斷重複舊內容。
- **互動流程更順暢**：提示處理、說話輪次、啟動行為與狀態回饋都經過調整，讓長對話更連貫，也更不容易出錯。同時也改善了空訊息處理與 Operator 啟動穩定性。
- **更高品質的 TTS**：更新後的 TTS 流程提升了語音品質，擴充了語言支援，並持續維持即時互動所需的速度。語音回退邏輯與非語音語言支援也有所改善，因此在無法輸出語音時體驗更平順。

### 性動作優化
本次更新也持續打磨了雙人程序化性動作，重點是讓動作更連貫、更容易理解，也更容易調整。這裡的許多改進都建立在 Sex Motion 3 功能頁描述的模型驅動接觸系統之上，也就是由女性端產生接觸幀，男性端綁定該接觸幀，讓雙方維持鎖定而不是逐漸漂移。

- **更真實的反應動作**
  反應動作現在能更自然地跟隨 thrust 延伸與身體運動。軀幹震動、脈動、阻尼與過衝控制都已細化，讓動作看起來更像自然反應，而不是刻意誇張，同時也更容易調出合適的脊椎反應。
- **高潮模擬**
  興奮、高潮 buildup 與回落行為已整理成更清楚的模型驅動邏輯。無論你想採用基於刺激的推進，或是更可控的節奏模式，現在都更容易調整；震動與表情強度層也更容易作為一個完整高潮段落來一起設定。
- **更準確的對齊與接觸求解**
  接觸計算加入了更好的接觸幀處理、髖部錨點對齊、自訂上方向、本地旋轉控制，以及更強的身體旋轉耦合。這些變更能在更多姿勢下維持雙人對齊，減少程序化動作常見的漂移感。
- **更多動作參數控制**
  速度、刺激、進度、回落以及快速動作設定都已整理優化。這讓你更容易調整節奏、自動化強度變化，並在柔和與強烈的動作風格之間切換，而不必和隱性的參數耦合對抗。
- **姿勢、視線與表情控制更細緻**
  隨機姿勢更新、身體彎曲、扭轉、頭部旋轉、強制注視與幀切換等功能，讓場景微調更靈活。眼神接觸、向下視線、眨眼、眼瞼控制與表情強度連動也都有改善，讓角色更有表現力，同時不顯得機械。

### 錯誤修正
- 優化了場景儲存邏輯，並改善了場景更新流程。
- 修正了多個 Operator 啟動與啟動穩定性問題。
- 改善了新聊天工作流程中的空訊息處理、端點命名、平台選擇，以及狀態或錯誤回饋。
- 整理了選單縮排、動作圖示處理等介面細節。
- 在整個程式碼庫中持續套用了多項空值安全與清理修正。
