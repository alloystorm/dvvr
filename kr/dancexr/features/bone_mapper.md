# 본 매퍼

본 본 매핑은 모델의 어느 본이 댄스XR의 표준 스켈레톤의 어떤 슬롯(머리, 등, 왼쪽 어깨, 오른쪽 검지 손가락 등)에 해당하는지 알려줍니다. 매핑이 완료되면, 모션이 모델을 구동할 수 있으며 댄스XR의 나머지 도구들도 올바르게 작동합니다.

본 본 매핑은 다음 모델에 적용됩니다:

- **XPS / XNALara** 모델 — XPS에는 표준 본 이름이 포함되어 있지 않으므로 필수입니다.
- **FBX** 모델 (프리뷰, 2025.9에 추가) — 동일한 이유로 필수입니다. FBX 본 이름은 소스마다 편차가 매우 큽니다.
- **PMX** 모델 — 보통 불필요합니다. PMX 파일에는 이미 표준 본 이름이 포함되어 있습니다.

모델을 로드했는데 동작 모션을 할당했음에도 불구하고 T-pose/표준 자세에 고정되어 있다면, 본 매핑 누락이 원인일 가능성이 높습니다.

## 매핑 상태

본 매퍼의 헤더는 누락된 본의 개수를 보여줍니다. 개수에 마우스를 올리면 상태 표시줄에 누락된 본의 이름을 볼 수 있습니다.

각 목표 본 옆에는 상태 아이콘이 있습니다:

- **빈 원** — 매핑되지 않았으나 선택 사항입니다. 모델은 이 본이 없어도 작동합니다.
- **점 있는 원** — 매핑됨.
- **느낌표가 있는 원 (!)** — 매핑되지 않았으며 **필수적**입니다. 이 본을 매핑하기 전까지 모델이 제대로 작동하지 않습니다.

대부분의 경우 댄스XR이 매핑을 자동으로 채워주었으며, `!`가 있는 몇몇 본에만 주의를 기울이면 됩니다.

## 자동 매핑

자동 매핑은 소스 모델의 이름을 댄스XR의 표준 스켈레톤과 일치시켜 본 할당을 채워줍니다. 2025.9에 개선되어 더 다양한 FBX 명명 규칙을 처리할 수 있게 되었으며, 수동으로 또는 [체인 매핑](#chain-mapping)을 통해 다듬을 수 있는 시작점이 됩니다.

**자동 매핑**을 클릭하거나 모델을 다시 로드하여 다시 실행합니다.

## 수동 매핑

댄스XR의 표준 스켈레톤에 있는 각 목표 본에 대해, 소스 모델에서 일치하는 본을 선택합니다. 자동 매핑이 잘못 처리한 본을 수정하거나, 알고리즘이 찾을 수 없었던 본을 매핑하는 데 사용하십시오.

수동 매핑은 원래의 매핑 모드이며, [체인 매핑](#chain-mapping)이 잘 처리하지 못하는 모든 경우에 여전히 사용할 수 있습니다. 데이터를 잃지 않고 수동 매핑과 체인 매핑 사이를 자유롭게 전환할 수 있습니다.

2026.6부터 비표준 본 변환 지원과 가용성 검사를 포함해 본과 리그 처리가 더 안정적으로 이루어져, 더 다양한 모델에서 DanceXR이 올바르게 작동하도록 돕습니다.

## 체인 매핑

2025.9에 추가되었습니다. 관련 본들을 체인으로 그룹화하고 좌우 대칭성을 활용하여 수동 매핑을 수행하는 더 빠른 방법입니다.

본 매퍼에서 **체인 매핑**을 활성화하여 이 모드로 전환하십시오. 전환한다고 해서 수동 매핑이 지워지지 않으며, 언제든지 비활성화하여 돌아갈 수 있습니다.

### 체인

본들은 부모-자식 계층 구조를 따르는 체인으로 그룹화됩니다. 머리를 찾기 위해 스켈레톤 트리의 각 레벨을 확장하는 대신, 평평한 상단-하단 목록을 보게 됩니다:

```
center → torso → torso2 → neck → head
```

각 행에 대해 해당하는 소스 본을 선택하면 부모-자식 경로가 자동으로 유지됩니다.

### 대칭 체인

대부분의 캐릭터 리그에는 좌우 쌍(왼팔 / 오른팔, 왼쪽 다리 / 오른쪽 다리)이 있습니다. 체인 매핑은 대칭 체인을 각각 하나의 행으로 병합합니다. 한쪽 측면을 매핑하면 댄스XR이 오른쪽 측면에도 할당을 미러링합니다. 작업은 한 번만 하면 됩니다.

### 전체 체인 한 번에 매핑하기

팔(어깨 → 위팔 → 아래팔 → 손 → 손가락)과 같은 대칭 체인의 경우, 소스 모델에서 **끝 본** (예: 검지 끝)을 선택하면 댄스XR이 부모 체인을 따라가면서 중간의 모든 본을 자동으로 채워줍니다.

이것은 손에 가장 빠른 경로입니다. 수동으로 다섯 손가락 리그를 매핑하는 것이 한쪽당 수십 개의 선택을 의미하는 것을 방지해 줍니다.

## 수동 매핑으로 되돌아가야 하는 경우

체인 매핑은 일반적인 경우를 처리합니다. 수동 매핑은 다음의 경우에 사용하십시오:

- 체인 가정을 깨는 비표준 부모-자식 관계를 가진 모델.
- 비대칭 리그(한 팔 캐릭터, 동일한 스켈레톤을 공유하는 소품).
- 댄스XR 표준 스켈레톤에는 존재하지만, 소스 모델에 명확한 대응본이 없는 본 — 가장 가까운 일치 항목을 직접 선택해야 합니다.

체인 모드로는 모델의 대부분을 매핑하고, 나머지 몇몇 본에 대해서만 수동 모드를 사용하십시오.
