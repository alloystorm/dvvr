# Release 2026.5

## DanceXR Operator 소개
DanceXR Operator는 DanceXR 전용 로컬 AI 백엔드입니다. 게임과 함께 로컬 서버로 실행되며, 음성 합성과 대규모 언어 모델 채팅을 하나의 통합 API 뒤에 묶어 별도의 클라우드 서비스에 의존하지 않고도 DanceXR에 AI 대화, 롤플레이, 캐릭터 음성 기능을 제공합니다.

실제로 이번 릴리스의 새로운 AI 채팅 스택을 뒷받침하는 기반이 바로 Operator입니다. 사용자의 게이밍 PC에서 직접 모델 관리와 음성 파이프라인을 로컬로 처리하도록 설계되어, 더 긴 대화와 더 장면 밀착형인 캐릭터 상호작용을 안정적으로 지원합니다.

## AI 채팅 개편
AI 채팅은 캐릭터 행동 시뮬레이션을 위한 agentic 워크플로를 중심으로 다시 구성되었습니다. Operator를 전용 로컬 백엔드로 사용하면서 별도의 서버 머신이나 클라우드 연결 없이도 일반적인 게이밍 환경에서 실행하기 쉬워졌습니다.

- **환경 인식**: 새로운 채팅 시스템은 장면 정보를 더 이른 시점에 읽고 더 일관되게 활용합니다. 그래서 캐릭터가 현재 위치, 시간대, 조명, 무대 구성 같은 요소를 반영해 반응할 수 있고, 맥락 없는 대사에서 벗어나기 쉬워졌습니다.
- **기억과 지속성**: 캐릭터는 최근 상호작용, 관계 맥락, 중요한 사건을 더 안정적으로 기억할 수 있게 되었습니다. 멀티턴 기록, 의도 추적, 세션 지속성, 메모리 압축이 함께 작동해 이전 대사를 반복하기보다 대화를 이어 가기 쉬워졌습니다.
- **상호작용 흐름 개선**: 프롬프트 처리, 화자 전환, 시작 동작, 상태 표시가 조정되어 긴 대화도 더 일관되고 덜 불안정해졌습니다. 빈 메시지 처리와 Operator 실행 안정성도 함께 좋아졌습니다.
- **고품질 TTS**: 업데이트된 TTS 파이프라인은 음성 품질을 높이고 지원 언어를 넓히면서도 실시간 상호작용에 필요한 속도를 유지합니다. 음성 출력이 불가능할 때의 폴백 동작과 비음성 언어 지원도 개선되어 전체 경험이 더 매끄러워졌습니다.

## 성행동 모션 개선
이번 릴리스에서는 절차형 페어 성행동 모션도 계속 다듬어졌습니다. 핵심은 모션이 더 연결되어 보이고, 더 읽기 쉽고, 더 쉽게 조정되도록 만드는 것입니다. 여기서의 많은 개선은 Sex Motion 3 기능 문서에 설명된 모델 기반 접촉 시스템, 즉 여성 측이 접촉 프레임을 만들고 남성 측이 그 프레임에 바인딩되어 서로 따로 떠다니지 않도록 하는 구조를 바탕으로 합니다.

- **더 자연스러운 반응 모션**
  반응 동작은 thrust 확장과 몸 움직임에 더 자연스럽게 따라가도록 조정되었습니다. 상체 흔들림, 박동감, 감쇠, 오버슈트 제어가 세밀해져 과장보다는 반응성 있는 움직임을 만들기 쉬워졌고, 척추 반응도 더 다듬기 쉬워졌습니다.
- **오르가즘 시뮬레이션**
  흥분, 피크까지의 buildup, 감쇠 동작은 더 명확한 모델 기반 로직으로 정리되었습니다. 자극 기반 진행과 의도적인 리듬 기반 진행 모두에서 더 예측 가능하게 제어할 수 있으며, 떨림과 표정 강도 레이어도 하나의 완성된 피크 단계처럼 조정하기 쉬워졌습니다.
- **정확한 정렬과 접촉 해결**
  접촉 계산에는 더 나은 접촉 프레임 처리, 힙 앵커 정렬, 사용자 지정 업 벡터, 로컬 회전 제어, 더 강한 몸 회전 결합이 추가되었습니다. 이 변화는 더 다양한 포즈에서도 페어 정렬을 유지하는 데 도움이 되고 절차형 모션의 떠 있는 느낌을 줄여 줍니다.
- **더 많은 모션 파라미터 제어**
  속도, 자극, 진행도, 감쇠, 빠른 모션 설정 주변이 정리되었습니다. 그 덕분에 템포를 조절하고 강도 변화를 자동화하거나, 부드러운 스타일부터 강한 스타일까지 전환할 때 숨은 설정 결합과 씨름할 필요가 줄었습니다.
- **포즈, 시선, 표정 제어 개선**
  랜덤 포즈 업데이트, 몸 굽힘, 비틀기, 머리 회전, 강제 바라보기, 프레임 이동 기능으로 장면 조정이 더 유연해졌습니다. 시선 고정, 아래 보기, 깜박임, 눈꺼풀 제어, 표정 강도 연동도 개선되어 캐릭터가 지나치게 기계적으로 보이지 않으면서 더 표현력 있게 반응합니다.

## 버그 수정
- 장면 저장 로직을 다듬고 장면 업데이트 흐름도 개선했습니다.
- 여러 Operator 실행 및 시작 안정성 문제를 수정했습니다.
- 새 채팅 워크플로의 빈 메시지 처리, 엔드포인트 이름, 플랫폼 선택, 상태 및 오류 보고를 개선했습니다.
- 메뉴 들여쓰기와 모션 아이콘 처리 같은 세부 UI 문제를 정리했습니다.
- 코드베이스 전반에 소규모 null 안전성 및 정리 수정을 계속 적용했습니다.
