# 保存场景

<!-- TODO: confirm exact UI flow. -->

保存的场景会捕获您 DanceXR 会话的**完整状态**——包括已加载的演员、动作、音频、舞台、灯光、摄影机、所有按演员设定的参数、所有环境设置——并将其保存到一个单个文件中，以便将来重新加载。

---

## 场景包含的内容

- 所有已加载的**演员**及其按演员配置（材质、物理、服装、动作覆盖）。
- 当前加载的**舞台**及其设置。
- 活动的**舞蹈套装**（音频 + 动作）以及分配给每个演员的设置。
- **环境** — 天空、地面、灯光、天气、水体。
- **摄影机**模式和设置。
- 多演员场景的**队形**布局。

场景**不**包含的内容：

- 实际的模型、动作或音乐文件。场景文件通过它们的内容库标识符引用它们（[自 2024.9](../releases/2024.9) 起，这是即时文件夹 + 文件名，而非完整路径，因此重新整理您的库不会破坏场景）。
- 接收者没有权限访问的层级特征。

---

## 保存和加载

1. 按您想要的设置配置场景。
2. 打开场景菜单，选择**保存场景**——命名场景文件。
3. 要重新加载，打开场景菜单，然后按名称选择保存的场景。

保存的场景存储在您的 [内容库](../preparecontent) 中，并且可以在设备之间复制。

---

## 场景无法完全加载时

如果一个场景引用了一个不存在于接收者库中的模型、动作或音乐文件，该资源将被跳过。场景的其余部分将正常加载。

要**将包含所有资源的场景一起分享**，请使用 [场景包](scene_bundle)——它会将模型和动作文件与场景文件一起打包，这样接收者就不必单独查找它们。

---

## 保存场景 vs 预设

| 格式 | 范围 | 页面 |
|---|---|---|
| **保存场景** | 场景中的所有内容 | (本页) |
| **场景包** | 保存场景 + 实际资源文件 | [场景包](scene_bundle) |
| **演员预设** | 仅单个演员的设置 | [演员预设](actor_presets) |
| **系统预设** | 仅场景范围的环境/图形设置 | [系统预设](system_presets) |
