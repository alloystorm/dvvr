# Release 2026.5

## 推出 DanceXR Operator
DanceXR Operator 是 DanceXR 的专用本地 AI 后端。它会作为本地服务器与游戏一起运行，把语音合成和大语言模型聊天整合在统一 API 后面，让 DanceXR 可以在不依赖额外云服务的情况下提供 AI 对话、角色扮演和角色语音功能。

在实际使用中，Operator 正是这次发布中全新 AI 聊天栈的基础。它被设计为直接运行在你自己的游戏 PC 上，在本地管理模型和语音流程，为更长时间、且更贴合场景的角色互动提供更可靠的底层支撑。

## AI 聊天重构
AI 聊天现已围绕角色行为模拟的 agentic 工作流重新构建。使用 Operator 作为专用本地后端后，它可以在标准游戏设备上运行，而不需要额外的服务器机器或云连接。

- **环境感知**：新的聊天系统会更早读取场景信息，也会更稳定地使用这些信息，因此角色能根据当前位置、时间、灯光、舞台配置等内容做出回应，而不是脱离场景地说话。
- **记忆与持续性**：角色现在能更稳定地记住最近互动、关系状态和关键事件。多轮历史、意图追踪、会话持久化和记忆压缩共同作用，让对话更容易延续，而不是不断重复旧台词。
- **交互流程更顺畅**：提示处理、说话轮次、启动行为和状态反馈都经过调整，让长对话更连贯，也更不容易出错。同时也改善了空消息处理和 Operator 启动稳定性。
- **更高质量的 TTS**：更新后的 TTS 流程提升了语音质量，扩展了语言支持，并继续保持实时互动所需的速度。语音回退逻辑和非语音语言支持也得到改进，因此在无法输出语音时体验更平滑。

## 性动作优化
本次更新也继续打磨了双人程序化性动作，重点是让动作更连贯、更容易读懂，也更容易调整。这里的很多改进都建立在 Sex Motion 3 功能页描述的模型驱动接触系统之上，也就是由女性侧生成接触帧，男性侧绑定该接触帧，从而让双方保持锁定，而不是逐渐漂移。

- **更真实的反应动作**
  反应动作现在能更自然地跟随 thrust 伸展和身体运动。躯干抖动、脉动、阻尼和过冲控制都经过细化，让动作更像真实反应，而不是刻意夸张，同时也更容易调出合适的脊柱响应。
- **高潮模拟**
  兴奋、高潮 buildup 和回落行为被整理成更清晰的模型驱动逻辑。无论你想要基于刺激推进，还是想要更可控的节奏模式，现在都更容易调节；抖动与表情强度层也更容易作为一个完整高潮阶段来统一调整。
- **更准确的对齐与接触求解**
  接触计算加入了更好的接触帧处理、髋部锚点对齐、自定义上方向、本地旋转控制，以及更强的身体旋转耦合。这些改动能在更多姿势下维持双人对齐，减少程序化动作中常见的漂移感。
- **更多动作参数控制**
  速度、刺激、进度、回落以及快速动作设置都经过整理。这让你更容易调节节奏、自动化强度变化，并在柔和与激烈的动作风格之间切换，而不用和隐性的参数耦合对抗。
- **姿势、视线与表情控制更细致**
  随机姿势更新、身体弯曲、扭转、头部旋转、强制注视和帧切换等功能，让场景微调更灵活。眼神接触、低头视线、眨眼、眼睑控制和表情强度联动也都有改进，让角色更有表现力，同时不显得机械。

## 错误修复
- 优化了场景保存逻辑，并改善了场景更新流程。
- 修复了多个 Operator 启动和启动稳定性问题。
- 改善了新聊天工作流中的空消息处理、端点命名、平台选择以及状态或错误反馈。
- 清理了菜单缩进、动作图标处理等界面细节。
- 在整个代码库中继续应用了多项空值安全与清理修复。
